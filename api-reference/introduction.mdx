## 快速开始

### 基础URL

```
REST API: https://api.trex.network/v1
RPC API: https://mainnet.trex.network
WebSocket: wss://mainnet.trex.network
```

### 认证

大部分API端点需要认证。使用API密钥进行认证：

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.trex.network/v1/endpoint
```

### 请求示例

```javascript
// REST API示例
const response = await fetch('https://api.trex.network/v1/blocks', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
```

```javascript
// RPC API示例
const response = await fetch('https://mainnet.trex.network', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    jsonrpc: '2.0',
    method: 'eth_blockNumber',
    params: [],
    id: 1
  })
});
```

## API端点

### 区块和交易

- `GET /blocks` - 获取区块列表
- `GET /blocks/:hash` - 获取区块详情
- `GET /transactions` - 获取交易列表
- `GET /transactions/:hash` - 获取交易详情

### 地址和账户

- `GET /addresses/:address` - 获取地址信息
- `GET /addresses/:address/balance` - 获取余额
- `GET /addresses/:address/tokens` - 获取代币余额

### 智能合约

- `GET /contracts/:address` - 获取合约信息
- `POST /contracts/:address/call` - 调用合约方法
- `GET /contracts/:address/events` - 获取合约事件

## 错误处理

API使用标准的HTTP状态码：

- `200` - 成功
- `400` - 请求错误
- `401` - 未认证
- `403` - 无权限
- `404` - 资源不存在
- `500` - 服务器错误

错误响应格式：

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Error description",
    "details": {}
  }
}
```

## 速率限制

API有速率限制以保护服务：

- **免费层**: 100请求/分钟
- **专业层**: 1000请求/分钟
- **企业层**: 自定义限制
